## 3. Sprint 3 기능 상세 기획

### Feature F-07: PDF 다운로드

**문제**: .docx 다운로드만 있어 일부 기기(모바일, iPad)에서 활용 어려움

**해결**: 브라우저 네이티브 인쇄 API를 활용한 PDF 저장 기능

**구현 방식**:
```
결과 페이지 → [PDF 저장] 버튼 클릭
  ↓
print-ready CSS 클래스 적용 (인쇄 영역 최적화)
  ↓
window.print() 호출 (브라우저 → PDF 저장 옵션)
```

**대안**: `@react-pdf/renderer` 라이브러리 (제어력 높지만 번들 크기 증가)
**채택**: 브라우저 print API (의존성 zero, 즉시 구현)

**구현 위치**: `src/app/generate/page.tsx`, `src/app/globals.css` (print CSS)

---

### Feature F-08: 교사 스타일 뱅크 (Teacher Style Bank, A-3)

**문제**: 현재 톤(academic/friendly/neutral)은 분위기만 조절, 실제 교사가 쓰는 문체 패턴 미반영

**해결**: 교사 문체 스타일 5가지 선택지 추가

**스타일 정의**:
```
1. 분석형 (Analytical)
   - "~을 분석하고 ~ 결론에 도달하였으며"
   - "데이터 기반 사고" 강조
   - 이과형 교사 스타일

2. 서사형 (Narrative)
   - "학생은 ~에 흥미를 느끼고 ~을 탐구하기 시작했다"
   - 학생 성장 스토리 중심
   - 문과/역사 교사 스타일

3. 역량 중심형 (Competency-based)
   - "비판적 사고력과 창의성을 발휘하여"
   - 핵심역량 키워드 명시
   - 입시 전문 교사 스타일

4. 간결형 (Concise)
   - 군더더기 없는 팩트 중심 서술
   - 명사형 종결 선호
   - 이공계 실험 교사 스타일

5. 격려형 (Encouraging)
   - "적극적으로 탐구에 임하며"
   - 학생 열정/노력 강조
   - 담임 교사 스타일
```

**UI**: 기존 톤 선택(3개) 옆에 "교사 스타일" 선택기(5개) 추가
**AI 반영**: 스타일별 프롬프트 instruction 추가 (lib/ai.ts)

**구현 위치**:
- `src/app/generate/page.tsx` — UI 선택기 추가
- `src/lib/ai.ts` — 스타일별 프롬프트 주입
- `src/types/index.ts` — TeacherStyle 타입 추가

### Feature F-09: 멀티 과목 배치 생성 (B-5)

**문제**: 학기말에 여러 과목 세특이 필요하나 1개씩만 생성 가능

**해결**: 홈 또는 별도 페이지에서 여러 과목+주제를 선택 후 한 번에 생성

**기능 흐름**:
```
[배치 생성 모드 진입]
  ↓
과목 1: 수학 + 주제 선택 + 옵션
과목 2: 물리 + 주제 선택 + 옵션
과목 3: (추가) ...  (최대 4개)
  ↓
[전체 생성 시작]
  ↓
병렬 생성 (Promise.all) → 진행바 표시
  ↓
과목별 탭으로 결과 표시
  ↓
전체 .docx 통합 다운로드 or 개별 복사
```

**API**: `POST /api/batch-generate` → 최대 4개 과목 병렬 생성
**제한**: 최대 4개 과목 (API 비용 및 응답 시간 고려)

**구현 위치**:
- `src/app/batch/page.tsx` — 신규 배치 생성 페이지
- `src/app/api/batch-generate/route.ts` — 신규 API
- `src/components/features/BatchSubjectSelector.tsx` — 과목+주제 선택 컴포넌트

## 4. 우선순위 및 구현 순서

```
F-07 (PDF) → F-08 (교사 스타일) → F-09 (배치 생성)
```

이유:
- F-07은 기존 결과 페이지에 버튼 추가 + CSS만 (30분)
- F-08은 생성 품질 직결, 기존 options 확장 (1.5시간)
- F-09은 신규 페이지 + API 신규 (3시간)

## 5. 아키텍처 검토 포인트

### 현재 아키텍처 리스크
1. **API 비용 증가**: 배치 생성 시 동시 호출 최대 4배 증가 → 요청당 rate limit 고려
2. **localStorage 한계**: 히스토리가 늘수록 용량 초과 가능성 → 최대 50개 제한 권고
3. **스트리밍 + 배치**: 배치 모드에서 스트리밍 UI는 복잡도 급증 → F-09는 비스트리밍으로 구현

### 기술 부채
- `InlineEditor.tsx` 설계서에 있으나 실제 구현 미확인 필요
- `generate/page.tsx`의 스트리밍 상태 관리 복잡도 증가 추세
