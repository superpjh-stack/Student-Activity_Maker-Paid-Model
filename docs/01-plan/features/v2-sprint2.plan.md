## 2. 기능 상세 기획

### Feature A-2: 대화형 탐구 코칭 플로우

**문제**: 현재 학생은 주제만 선택하면 끝 → 탐구 경험이 없는 빈 보고서가 생성됨

**해결**: 주제 선택 후 3가지 코칭 질문으로 학생 경험을 수집 → 생성 시 반영

**코칭 질문 흐름**
```
주제 선택 완료
    ↓
"이 주제를 선택한 이유가 있나요?"
  [자유 입력 / 건너뛰기]
    ↓
"이 주제와 관련해 직접 해본 활동이 있나요?"
  [자유 입력 / 건너뛰기]
    ↓
"탐구를 통해 무엇을 알고 싶나요?"
  [자유 입력 / 건너뛰기]
    ↓
생성 요청 시 답변 → 프롬프트에 자동 주입
```

**구현 위치**: `/[subject]/page.tsx` 또는 `/generate/page.tsx` 주제 확정 직후

**사용자 UX 원칙**
- 모든 질문은 선택사항 (건너뛰기 가능)
- 입력한 내용은 보고서 동기/활동 파트에 자연스럽게 녹아들어야 함
- UI: 카드 슬라이드 방식 (1문답씩 순차 표시)

---

### Feature B-1: 마인드맵 탐구 주제 확장기

**문제**: 현재 주제는 과목별 고정 5개 → 창의적 탐구 동기 부족

**해결**: 홈 또는 주제선택 화면에 "키워드로 주제 탐색" 기능 추가

**기능 흐름**
```
[홈 또는 주제선택 페이지]
    ↓ "키워드로 탐구 주제 찾기" 버튼
[키워드 입력 화면]
  예: "기후변화", "인공지능", "운동"
    ↓
[AI 주제 확장 결과 - 그리드 표시]
  과목별로 연계된 탐구 주제 카드 (최대 16개)
  예: 수학 - "기후 데이터 회귀분석"
      물리 - "탄소 포집 장치 원리"
      사회 - "기후 정의 국제협약 분석"
    ↓
  카드 클릭 → 해당 과목 + 주제로 바로 생성 시작
```

**API**: `POST /api/expand-topics` → AI가 키워드 기반 과목별 주제 생성

**표시 방식**: 과목별 색상 카드 그리드 (이미 있는 과목 색상 재활용)

---

### Feature B-2: AI 내 글 피드백

**문제**: 학생이 직접 쓴 세특/보고서를 검토할 방법이 없음

**해결**: 새 페이지 `/feedback` — 텍스트 붙여넣기 → AI 점수 + 개선 제안

**피드백 항목**
- 분량 적절성 (세특: 500자, 보고서: 선택한 길이 기준)
- 구조 완성도 (동기 → 활동 → 결과 → 진로연계 4단계 포함 여부)
- 표현 자연성 (어색한 표현, 반복 표현 지적)
- 교사 관점 리뷰 (실제 교사가 쓸 법한 문체인가)
- 개선 제안 + 수정 버전 제공

**진입 경로**
- 헤더/메뉴에 "내 글 피드백" 메뉴 추가
- 결과 페이지에 "내 글 가져와서 검토하기" 링크 추가

## 3. 우선순위 및 구현 순서

```
A-2 (대화형 코칭) → B-1 (주제 확장기) → B-2 (AI 피드백)
```

이유:
- A-2는 기존 흐름에 자연스럽게 삽입 (가장 많은 사용자에게 노출)
- B-1은 신규 진입점 창출 (사용자 획득)
- B-2는 독립 기능 (별도 페이지, 다른 기능에 영향 없음)

---

## 4. 기술 방향 (아키텍트 팀 전달)

- **A-2**: 클라이언트 state만으로 구현 가능 (localStorage or URL params)
- **B-1**: `POST /api/expand-topics` 신규 API + 결과 그리드 컴포넌트
- **B-2**: `POST /api/feedback` 신규 API + `/feedback` 신규 페이지
