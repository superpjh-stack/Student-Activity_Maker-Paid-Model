# Plan: 내 이야기 담기 — 진정성 있는 탐구 경험 표현 시스템

> 작성일: 2026-03-05
> CTO팀: CTO(조율) + 기획자 A(UX/경험) + 기획자 B(AI/기술) + 기획자 C(제품/우선순위)
> Feature ID: authentic-experience

---

## 1. 문제 정의

### 핵심 통증 (Core Pain)

> "AI가 한 것 같다" — 탐구보고서와 세특에 학생의 실제 경험이 0% 담겨 있다.

**현재 흐름**:
```
과목 선택 → 주제 선택 → AI가 전부 씀 → 복사
```

**문제의 본질**: AI는 해당 주제의 '전형적인 탐구 보고서'를 쓴다. 어느 학생이 써도 같은 내용.
- "전자기 유도 현상을 탐구하였다" → 누구나 쓸 수 있는 문장
- 학생의 실제 경험, 실수, 발견, 깨달음이 없다
- 입학사정관은 이 차이를 즉시 알아챈다

### 목표 상태 (Target State)

```
과목 선택 → 주제 선택 → 🎤 내 경험 한땀한땀 입력 → AI가 내 이야기로 엮어줌 → 내가 쓴 것 같은 결과
```

## 2. 핵심 인사이트 (CTO팀 합의)

**"학생은 이미 경험을 가지고 있다. AI가 물어봐야 한다."** (기획자 A)

**"수집한 경험 단편을 AI 프롬프트의 필수 앵커로 만들면 된다."** (기획자 B)

**"기존 코칭 모달을 진화시키면 리스크 없이 빠르게 갈 수 있다."** (기획자 C)

→ **결론**: 경험 수집 UI + 경험 앵커 프롬프팅 조합으로 "진정성 있는 AI 생성" 달성

## 3. 기능 상세 기획

### F-15: 경험 인터뷰 시스템 (My Story Modal) ⭐ 최우선

**현재 코칭 모달 (3필드) → 내 이야기 모달 (5필드)로 진화**

| 필드 | 라벨 | 힌트 텍스트 | 기존 여부 |
|------|------|-------------|-----------|
| motivation | 💡 이 주제를 선택한 계기 | "수업 중 어떤 순간이었나요? 유튜브, 뉴스, 일상의 궁금증?" | 기존 (확장) |
| activity | 🔬 실제 해본 활동 | "직접 실험, 조사, 계산, 인터뷰 등 뭔가 해봤나요?" | 기존 유지 |
| discovery | 😲 의외의 발견 / 놀란 점 | "예상과 달랐던 점, 처음엔 틀렸던 가설, 뜻밖의 결과?" | **신규** |
| curiosity | 🤔 더 알고 싶은 것 | "이 탐구 후 새로 생긴 질문은?" | 기존 유지 |
| change | 🌱 생각·진로의 변화 | "이 탐구가 당신의 생각이나 진로 방향을 어떻게 바꿨나요?" | **신규** |

**UX 원칙**:
- 모든 필드 선택사항 (강제 입력 없음)
- 채울수록 진정성 게이지(F-17) 올라감
- 각 필드에 예시 힌트 버튼 ("예시 보기") → 클릭하면 구체적 예시 팝업

**진입점**: 생성 페이지 → 기존 "코칭 메모" 버튼을 "내 이야기 담기 ✍️" 버튼으로 리브랜딩

---

### F-16: 경험 앵커 프롬프팅 (Experience Anchor Prompting) ⭐ 최우선

**AI 프롬프트 내 경험 단편 필수 삽입 시스템**

```
입력된 경험 단편 → 프롬프트에 '필수 앵커'로 주입 → AI가 반드시 녹여서 생성
```

**프롬프트 전략**:

```
[학생의 실제 경험 단편 — 절대 생략·변경 금지]
다음 내용을 보고서/세특에 반드시 자연스럽게 녹여서 작성할 것:

① 탐구 계기 (학생이 직접 경험한 구체적 순간으로 서술):
   "{motivation 입력값}"

② 탐구 활동 중 발견 (학생 관점의 놀람/발견으로 서술):
   "{discovery 입력값}"

③ 깨달음과 변화 (1인칭 성장 서사로 마무리):
   "{change 입력값}"

⚠️ 위 세 경험은 학생의 고유한 기억이다.
   일반적인 표현으로 대체하지 말고 원문의 의미를 살려서 구체적으로 녹여 쓸 것.
```

**기대 결과 비교**:

| 구분 | 기존 | 경험 앵커 적용 후 |
|------|------|-----------------|
| 도입부 | "전자기 유도에 관심을 가지고 탐구를 시작하였다." | "물리 수업 중 코일 실험을 하다가 예상보다 훨씬 강한 전류가 측정되어 의아함을 느낀 것이 계기가 되었다." |
| 중간부 | "실험 결과 코일 수에 따라 전류가 변화하였다." | "처음에는 코일 수와 전류가 선형 관계일 것이라 예상했으나, 실제로는 비선형적으로 증가하는 결과를 마주하며 새로운 의문이 생겼다." |
| 결론 | "이 탐구를 통해 전자기 유도의 원리를 이해하였다." | "이 탐구 이후 에너지 변환의 복잡성에 매료되어 에너지공학 분야 진로를 더 真剣히 고민하게 되었다." |

### F-17: 진정성 게이지 UI (Authenticity Meter)

**목적**: 학생이 경험을 더 많이 입력할수록 시각적 피드백을 줘서 참여 유도

**UI 구현**:
```
내 이야기 반영도
[████████░░] 80%
✅ 탐구 계기 · ✅ 실제 활동 · ✅ 의외의 발견 · ✅ 변화
⬜ 더 알고 싶은 것

"입력 필드를 채울수록 AI가 '내가 쓴 것 같은' 결과를 만들어요"
```

**게이지 계산**:
- 5개 필드 각 20점
- 10자 미만: 5점 (있기는 한데 빈약)
- 30자 이상: 20점 (충분)
- 총점 → 퍼센트로 표시

**게이지 레벨별 메시지**:
| 퍼센트 | 메시지 | 색상 |
|--------|--------|------|
| 0~20% | "AI 기본 생성 모드" | 회색 |
| 21~50% | "조금 더 나다워요" | 노랑 |
| 51~80% | "내 이야기가 담겼어요" | 초록 |
| 81~100% | "완전히 내 이야기예요 ✨" | 보라 |

---

### F-18: 인간화 문체 모드 (Human Writing Mode)

**목적**: 경험 입력과 무관하게 AI 생성 문체 자체를 덜 'AI스럽게' 만드는 옵션

**인간화 문체 특징**:
```
- "처음에는 ~인 줄 알았는데" 형식
- "~이 의외로 ~했다", "놀랍게도 ~"
- 완벽하지 않은 탐구 과정 표현 ("시행착오 끝에", "여러 번 시도한 결과")
- 구체적 숫자/수업명/책 이름 등 세부 맥락
- 결론 전 주저함: "확신하기는 어렵지만", "더 연구가 필요하지만"
```

**구현**: 기존 교사 스타일 뱅크 옆에 "✍️ 인간적 문체" 토글 추가

---

### F-19: 경험 구절 하이라이트 (Experience Highlight) — 3순위

**목적**: 생성 결과에서 "학생의 실제 경험이 반영된 부분"을 시각적으로 표시

**UX**:
- 생성 완료 후 상단에 "✨ 내 이야기가 반영된 구절" 섹션
- 경험 앵커에서 파생된 문장들을 노란색/보라색으로 하이라이트
- 클릭하면 어떤 입력에서 왔는지 연결 표시

## 4. 구현 우선순위 및 의존성

```
F-16 (앵커 프롬프팅) ← 가장 빠름, ai.ts 수정만
    ↓
F-15 (내 이야기 모달) ← 기존 CoachingModal 확장
    ↓
F-17 (진정성 게이지) ← F-15 완성 후 UI 추가
    ↓
F-18 (인간화 문체) ← 독립적, 빠르게 추가 가능
    ↓
F-19 (경험 하이라이트) ← 후순위, 복잡도 있음
```

**예상 구현 시간**:
| 기능 | 예상 시간 |
|------|---------|
| F-16 앵커 프롬프팅 | 30분 (ai.ts 프롬프트 수정) |
| F-15 내 이야기 모달 | 1.5시간 (CoachingModal 확장) |
| F-17 진정성 게이지 | 1시간 (컴포넌트 신규) |
| F-18 인간화 문체 | 30분 (토글 + 프롬프트 추가) |
| F-19 경험 하이라이트 | 2시간 (파싱 로직 복잡) |
