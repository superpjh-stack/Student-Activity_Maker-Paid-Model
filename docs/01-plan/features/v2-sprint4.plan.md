## 2. Sprint 4 기능 상세 기획

### Feature F-10: SEO 최적화 + 랜딩 페이지 개편

**현황**: 홈페이지가 단순 앱 진입 화면 → 디자인 리뉴얼 이미 진행 중(globals.css 개편)

**추가 작업**:
```
1. Next.js Metadata API 적용
   - title: "AI 생기부 Maker — 세특·탐구보고서 5분 완성"
   - description: 학생 키워드 중심 120자
   - openGraph: og:title, og:description, og:image (동적 생성)
   - keywords: 세특, 생기부, 탐구보고서, AI 세특 생성

2. 구조화 데이터 (JSON-LD)
   - WebApplication schema
   - FAQ schema (자주 묻는 질문 섹션 추가)

3. robots.txt + sitemap.xml 생성
   - 검색엔진 크롤링 허용
   - 주요 경로 sitemap 등록

4. 홈페이지 FAQ 섹션 추가
   - "세특이 뭔가요?", "AI가 생성한 세특을 제출해도 되나요?" 등
   - SEO long-tail 키워드 자연스럽게 포함
```

**구현 위치**:
- `src/app/layout.tsx` — Metadata 추가
- `src/app/page.tsx` — FAQ 섹션, 랜딩 강화
- `public/robots.txt`, `src/app/sitemap.ts` 신규

### Feature F-11: 인라인 에디터 (Inline Editor)

**문제**: 생성 완료 후 세특/보고서를 수정하려면 외부 텍스트 에디터에서 편집 후 다시 붙여넣어야 함

**해결**: 결과 페이지에서 텍스트를 바로 클릭해서 수정 가능

**UX 흐름**:
```
[생성 완료]
  ↓
결과 텍스트 우측 상단: [수정하기] 버튼
  ↓ 클릭
textarea 전환 (현재 텍스트 pre-filled)
  ↓
수정 후 [저장] or [취소]
  저장: 로컬 상태 업데이트, 이력 자동 재저장
  취소: 원본 복원
  ↓
수정된 버전으로 [복사] / [PDF 저장] / [.docx 저장]
```

**세특 전용**: 수정 중 실시간 글자수 카운터 표시 (500자 기준 ±10자 배지)

**구현 위치**:
- `src/components/features/ResultDisplay.tsx` — 인라인 에디터 통합
- 별도 컴포넌트: `src/components/features/InlineEditor.tsx`

### Feature F-12: 포트폴리오 생성기 (C-3)

**문제**: 이력에 쌓인 세특들이 개별 항목으로만 존재, 입시 포트폴리오로 활용 불가

**해결**: 이력 페이지에서 세특 여러 개를 선택 → AI가 하나의 일관된 포트폴리오 문서로 엮어줌

**생성 결과물 구조**:
```
[학생 포트폴리오]
━━━━━━━━━━━━━━━━━━━━━

1. 학습 여정 요약
   "이 학생은 [주제A], [주제B], [주제C]를 탐구하며
    일관된 [관심 분야] 역량을 보여주었습니다."

2. 과목별 탐구 활동 요약 (세특 기반)
   - 수학: 주제 + 핵심 탐구 내용 1-2줄 요약
   - 물리: ...

3. 탐구 활동의 공통 주제 / 역량
   AI가 공통 스레드를 뽑아 서사화

4. 진로 연계 스토리
   "위 탐구 활동들은 [희망 진로]와 연결되며..."

━━━━━━━━━━━━━━━━━━━━━
출력: docx + PDF 저장 가능
```

**UI 흐름**:
```
/history 페이지
  ↓ 세특 항목 체크박스로 선택 (1~6개)
  ↓ [포트폴리오 생성] 버튼
/portfolio 페이지
  ↓ AI 생성 (스트리밍)
  ↓ 결과 표시 + docx/PDF 다운로드
```

**API**: `POST /api/generate-portfolio`
**입력**: 선택한 이력 항목들 + 선택적 학생 프로필

**구현 위치**:
- `src/app/portfolio/page.tsx` — 포트폴리오 결과 페이지
- `src/app/api/generate-portfolio/route.ts` — 신규 API
- `src/app/history/page.tsx` — 체크박스 선택 UI 추가
- `src/lib/ai.ts` — `generatePortfolio()` 함수 추가

### Feature F-13: PWA 모바일 최적화

**문제**: 현재 앱이 모바일 브라우저에서만 접근 가능, 홈 화면 추가 불가

**해결**: Next.js PWA 설정으로 설치형 앱 경험 제공

**구현 내용**:
```
1. next-pwa 패키지 설정
   - Service Worker 자동 생성
   - 오프라인 시 기본 캐시 페이지 표시

2. manifest.json 추가
   - 앱 이름: "AI 생기부"
   - 아이콘: 512x512 (현재 로고 기반)
   - display: "standalone"
   - theme_color: "#7c3aed"

3. 터치 UX 개선
   - 버튼 최소 터치 영역 44px
   - 모바일에서 텍스트 영역 줄임
   - 과목 카드 터치 피드백 개선
```

**구현 위치**: `next.config.ts`, `public/manifest.json`, `public/icons/`

### Feature F-14: 진로 로드맵 추천 (B-3)

**문제**: 탐구 완료 후 "다음에 뭘 해야 하나" 방향성 없음

**해결**: 생성 결과 하단에 "이 탐구와 연계된 다음 주제 3가지" AI 추천 카드

**추천 로직**:
```
완료된 탐구 정보 (과목 + 주제 + 세특 내용)
  ↓
AI 분석: 탐구의 핵심 개념, 연계 분야
  ↓
추천 결과: 3개 카드
  [과목 뱃지] 수학
  "회귀분석으로 기후 데이터 예측 모델링"
  → [이 주제로 탐구 시작] 버튼
```

**진입점**: 생성 결과 페이지 맨 아래 "다음 탐구 추천" 섹션

**API**: `POST /api/recommend-topics`
**구현 위치**:
- `src/app/api/recommend-topics/route.ts` — 신규
- `src/components/features/NextTopicRecommend.tsx` — 신규

```
F-10 (SEO/랜딩) → F-11 (인라인 에디터) → F-12 (포트폴리오) → F-14 (진로 추천) → F-13 (PWA)
```

- F-14: 결과 페이지 확장, 가벼운 추가 (1시간)
- F-13: 설정 위주, 아이콘 준비 필요 (1시간)

## 4. 기술 아키텍처 방향

### 포트폴리오 AI 프롬프트 전략
- 입력: 최대 6개 세특 전문 + 학생 프로필
- GPT-4o 128k 컨텍스트 충분 활용
- 출력: 섹션 구조화된 포트폴리오 텍스트 (마크다운)

### SEO
- Next.js 14 App Router Metadata API (서버 컴포넌트)
- 홈 페이지를 Server Component로 전환 필요 (현재 'use client')
- 동적 OG 이미지: `/api/og` route (next/og 활용)

### PWA
- `next-pwa` 또는 `@ducanh2912/next-pwa` 패키지
- Cloud Run 배포 환경에서 서비스워커 HTTPS 필수 → 이미 충족
