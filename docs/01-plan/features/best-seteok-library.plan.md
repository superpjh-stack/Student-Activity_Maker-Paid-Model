## 1. 서비스 개요

### 한 줄 정의
> 우수한 세특·탐구보고서를 학생이 직접 기증하고, AI가 이를 RAG로 참조하여 더 높은 퀄리티의 세특을 생성하는 크라우드소싱 기반 참조 서비스

### 핵심 가치 제안

| 대상 | 가치 |
|------|------|
| 기여자 (등록자) | "내 우수 세특이 후배들에게 도움이 된다" + 크레딧 보상 |
| 소비자 (사용자) | "합격생이 쓴 방식으로 내 세특을 만들 수 있다" |
| 플랫폼 | 사용자 증가 → 라이브러리 풍부 → 품질 향상 → 사용자 증가 (선순환) |

### 기존 서비스 대비 차별화

| 비교 항목 | 기존 | 추가 후 |
|-----------|------|---------|
| AI 참조 데이터 | 일반 프롬프트 | 실제 학생 우수 사례 기반 RAG |
| 결과물 수준 | AI 표준 퀄리티 | 합격생 수준 참조 가능 |
| 경쟁 진입 장벽 | 낮음 | 높음 (DB 1,000건+ 시 복제 불가) |
| 구독 전환 트리거 | 사용량 제한 | 참조 기능 잠금 해제 |

## 3. 사용자 여정

### 3-1. 기여자 (등록자) 여정

**주요 페르소나**

| 페르소나 | 특성 | 동기 | 주요 장벽 | 해결책 |
|---------|------|------|-----------|--------|
| 합격생 | 수능/수시 종료 | 후배 기여 욕구 + 합격 스토리 인정 | 귀찮음, 개인정보 노출 | 3분 등록 + 자동 마스킹 |
| 성적 상위 재학생 | 자신감 있는 고2~3 | "내가 레퍼런스가 된다" + 크레딧 | "완성형이 아닌데?" | "초안도 가능" 명시 |
| 선생님/강사 | 우수 사례 보유 | 학생 참고 자료 구조화 | 학생 동의 문제 | 동의 체크박스 필수 UI |

**3단계 등록 플로우**

```
Step 1: 메타데이터 입력 (1분)
  → 종류(세특/탐구보고서), 과목, 학년, 진로, 합격 여부, 공개 레벨

Step 2: 본문 입력 + 자동 마스킹 (1분 30초)
  → 이름·학교명·지역·교사명·연도 자동 감지 및 치환
  → Before/After 나란히 미리보기

Step 3: AI 검수 결과 + 동의 (30초)
  → 품질점수 표시, 개인정보 마스킹 확인
  → 비독점 영구 라이선스 동의 체크
  → 예상 획득 크레딧 표시
```

**공개 레벨 3단계**

| 레벨 | 공개 | 비공개 |
|------|------|--------|
| 완전 익명 (기본) | 과목, 학년, 진로, 품질점수 | 학교, 이름, 대학명, 원문 |
| 부분 공개 | + 합격 대학군(SKY/상위권 수준) | 학교명, 이름 |
| 합격 인증 공개 | + 합격 대학명 (선택) | 이름, 학교명 |

> **핵심 원칙**: 원문 전체는 어떤 경우에도 외부 공개 금지. AI만 참조.

### 3-2. 소비자 (사용자) 여정

**진입점 3곳**

1. 주제 선택 후 자동 노출: "이 주제로 Best 사례 N건이 있어요" 팝업 배너
2. 생성 설정 내 옵션 섹션 (기본: 접힘)
3. 별도 `/library` 탐색 페이지

**참조 방식 3종**

| 모드 | AI 지시 | 결과 특성 | 구독 |
|------|---------|----------|------|
| 구조만 참조 | 단락 구성·논리 흐름만 따르되 내용 새로 작성 | 구조 유사, 내용 독창 | Standard+ |
| 스타일만 참조 | 문장 길이·전문 용어 밀도·감성 비율만 참조 | 문체 유사, 구조 독창 | Standard+ |
| 구조+스타일 | 구조+스타일 종합 참조, 사실 관계는 새로 생성 | 최고 퀄리티 출력 | Premium 전용 |

**표절 방지 안전장치**

- 프롬프트: "5단어 이상 연속 동일 문구 금지" 명시
- 생성 후: 참조 원문 대비 유사도 20%+ 시 자동 재생성
- 결과물: "참조 기반 생성" 표시 (선택적 제거 가능)

## 4. AI 시스템 설계

### 4-1. 기술 방식 결정: RAG 채택

| 방식 | 결정 | 이유 |
|------|:----:|------|
| RAG (pgvector) | **채택** | 즉시 구현, 비용 최적, 실시간 콘텐츠 반영 |
| Fine-tuning | 배제 | 수천 건 필요, 재학습 주기 부담, 고비용 |
| Prompt Engineering만 | 배제 | 차별화 불충분 |

### 4-2. 임베딩 전략

- **모델**: `text-embedding-3-small` (1,536차원)
- **저장**: 기존 Supabase PostgreSQL + `pgvector` 확장 (별도 벡터DB 불필요)
- **인덱스**: IVFFlat (lists=100)
- **검색 순위 가중치**:
  - 코사인 유사도 × 0.7
  - AI 품질점수 × 0.2
  - 최신성 × 0.1
- **품질 게이트**: 품질점수 70점 미만은 RAG 풀 자동 제외

## 5. 데이터 모델 요약

```sql
CREATE TABLE best_seteok (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  contributor_id  UUID REFERENCES users(id),
  type            TEXT CHECK (type IN ('seteok', 'report')),
  subject         TEXT NOT NULL,
  grade           SMALLINT CHECK (grade IN (1,2,3)),
  career_field    TEXT,
  admission_tier  TEXT CHECK (admission_tier IN ('sky','top','none')),
  content         TEXT NOT NULL,          -- 마스킹된 본문 (원문 비공개)
  embedding       vector(1536),           -- pgvector
  quality_score   SMALLINT DEFAULT 0,     -- 0~100
  ref_count       INTEGER DEFAULT 0,
  editor_pick     BOOLEAN DEFAULT false,
  visibility      TEXT DEFAULT 'anonymous',
  status          TEXT DEFAULT 'pending', -- pending/approved/rejected/archived
  created_at      TIMESTAMPTZ DEFAULT now(),
  updated_at      TIMESTAMPTZ DEFAULT now()
);

-- 성능 인덱스
CREATE INDEX ON best_seteok USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
CREATE INDEX ON best_seteok (subject, grade, status);
CREATE INDEX ON best_seteok (ref_count DESC) WHERE status = 'approved';
```

## 7. 품질 보장 시스템

### AI 자동 검수 기준 (등록 즉시 ~5초)

| 항목 | 기준 | 결과 |
|------|------|------|
| 분량 | 세특: 400~600자, 탐구: 500자+ | 미달 시 안내 |
| 개인정보 | 이름·학교·전화·주민번호 패턴 | 마스킹 후 재확인 |
| 품질점수 | 60점 이상 | 미달 시 등록 불가 |
| 언어 | 한국어 80%+ | 미달 시 반려 |
| 중복 | 기존 DB와 유사도 70%+ | 수동 검수 큐 이동 |
| 부적절 | 혐오·광고·허위 | 즉시 반려 |

### 수동 검수 (24시간 이내, 플래그 케이스만)
- 유사도 70%+ 감지, 합격 인증 요청, 신고 접수
- 관리자 대시보드: 승인 / 반려 / 수정 요청

### 자동 마스킹 규칙

| 대상 | 패턴 | 치환 |
|------|------|------|
| 실명 | 2~4자 한국어 이름 | "본인" |
| 학교명 | "고등학교" 포함 | "재학 학교" |
| 지역명 | 강남·대치동 등 | 삭제 |
| 담당 교사명 | 교사/선생님 앞 이름 | "담당 교사" |
| 특정 연도 | 입시 시즌 유추 가능 연도 | "해당 연도" |

## 8. 게임화 & 사회적 증거

### 랭킹 시스템
- 월간 Most Referenced Top 5 (메인 라이브러리 페이지)
- 기여자 개인 기여 현황 패널 (누적 참조 수, 획득 크레딧, 배지)

### 배지

| 배지 | 조건 |
|------|------|
| 기여자 | 첫 등록 완료 |
| 인기 기여자 | 누적 참조 100회+ |
| 합격 인증 | 합격 대학 인증 완료 |
| Top 10 기여자 | 월간 참조 수 상위 10명 |

### 사회적 증거 문구 전략
- "이 사례를 참조한 학생 N명이 생성 완료" (생기부 카드)
- "이 주제로 참조 생성한 학생이 이번 달 N명 있었어요" (생성 완료 페이지)
- **주의**: "참조해서 합격"이라는 인과 관계 표현 사용 금지

## 10. 법적·윤리 검토

### 저작권

- 등록 시 명시적 동의 필수: "플랫폼이 AI 참조 목적으로 사용하는 것에 동의"

- AI 자동 마스킹 + 편집자 2차 확인

- "입시 성공한 학생이 기증한 생기부" 표현으로만 사용
